Semantic Elements

What are they and why do we care?

What does "semantics" mean?

Before we can understand what HTML semantic elements are, we need to understand what semantics means. Merriam-Webster defines semantics* as "the study of meanings". A more detailed description is the historical and psychological study and the classification of changes in the signification of words or forms viewed as factors in linguistic development.

So what does that mean?

This simply means words have meaning. For instance, curse words have a negative connotation because humans give them a negative connotation. We could just as easily use the word "fork*" as a four-letter curse word but we don't because we want "fork" to mean an implement with two or more prongs used especially for taking up (as in eating), pitching, or digging.

Okay, then what are semantic elements in HTML?

Semantic elements have changed the way we code in HTML. HTML elements used to provide direction for how the page would be presented or how it would look, but offered no real meaning. Semantic HTML tags actually say what they mean. We can use elements such as: header, body, aside, and figure to divide our content and provide meaning to various sections. You may ask why this is important: doesn’t dividing the content using the old div element accomplish the same thing? The simple answer is yes. However in language we give words meanings. These meanings ensure everyone understands that when we go into a store to buy a dining “table*” we are looking for a smooth flat slab fixed on legs. By using semantic elements that actually say what we mean (such as header or video), we can all understand what is being accomplished in the code.

What are the benefits of semantic elements?

Besides the obvious benefit of elements having meaning, using known semantic elements allows for:

  • browser support across all modern browsers,
  • programs and applications to understand your page,
  • improved SEO,
  • developers to understand their and other's code without additional effort.

What are some examples of semantic elements in HTML?

Note: This is not a complete list of semantic elements and is only meant to show how explicit semantic elements are.

Element Definition
header represents introductory content, typically a group of introductory or navigational aids
nav represents a section of a page whose purpose is to provide navigation links, either within the current document or to other documents
style contains style information for a document, or part of a document
aside represents a portion of a document whose content is only indirectly related to the document's main content
blockquote indicates that the enclosed text is an extended quotation
strong indicates that its contents have strong importance, seriousness, or urgency. Browsers typically render the contents in bold type.

*Definitions for semantics, fork, and table courtesy of: Merriam Webster
